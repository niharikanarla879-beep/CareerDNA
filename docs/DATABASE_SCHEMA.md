# Database Schema - CareerDNA

This document outlines the PostgreSQL database schema for the CareerDNA platform. The schema is defined here conceptually and as a Prisma ORM schema representation.

---

## 2. Table Specifications

### 2.1. `User`
Stores core user credentials and system access tokens.
- `id` (UUID, Primary Key)
- `email` (VARCHAR, Unique, Indexed)
- `passwordHash` (VARCHAR, Nullable for OAuth accounts)
- `createdAt` (TIMESTAMP, default: now)
- `updatedAt` (TIMESTAMP, default: now)

### 2.2. `Profile`
Contains user-facing bio details and professional summaries.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`, Unique)
- `firstName` (VARCHAR)
- `lastName` (VARCHAR)
- `avatarUrl` (VARCHAR, Nullable)
- `currentJob` (VARCHAR, Nullable)
- `educationLevel` (VARCHAR, Nullable)
- `experienceYears` (INTEGER, default: 0)
- `updatedAt` (TIMESTAMP, default: now)

### 2.3. `UserSkill`
Holds a list of skills mapped to the user, either declared manually or parsed from a resume.
- `id` (UUID, Primary Key)
- `profileId` (UUID, Foreign Key referencing `Profile.id`)
- `name` (VARCHAR, Indexed)
- `level` (ENUM: 'BEGINNER', 'INTERMEDIATE', 'ADVANCED')
- `source` (ENUM: 'MANUAL', 'RESUME_PARSED')

### 2.4. `Project`
Logs user portfolio projects, links, and strength score.
- `id` (UUID, Primary Key)
- `profileId` (UUID, Foreign Key referencing `Profile.id`)
- `title` (VARCHAR)
- `description` (TEXT)
- `githubUrl` (VARCHAR, Nullable)
- `demoUrl` (VARCHAR, Nullable)
- `techStack` (TEXT[])
- `strengthScore` (INTEGER, default: 0) - Calculated based on description, git activity, and links.
- `createdAt` (TIMESTAMP, default: now)
- `updatedAt` (TIMESTAMP, default: now)

### 2.5. `Certification`
Tracks user certifications and progress.
- `id` (UUID, Primary Key)
- `profileId` (UUID, Foreign Key referencing `Profile.id`)
- `title` (VARCHAR)
- `issuer` (VARCHAR)
- `issueDate` (TIMESTAMP, Nullable)
- `credentialId` (VARCHAR, Nullable)
- `credentialUrl` (VARCHAR, Nullable)
- `relevanceScore` (INTEGER, default: 0) - Strong, Medium, Weak relative to target career path.
- `status` (ENUM: 'NOT_STARTED', 'IN_PROGRESS', 'COMPLETED', default: 'COMPLETED')
- `createdAt` (TIMESTAMP, default: now)
- `updatedAt` (TIMESTAMP, default: now)

### 2.6. `AssessmentResult`
Saves the results of the Holland Code (RIASEC), values inventory, and general interests.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`)
- `riasecScores` (JSONB): e.g., `{ R: 12, I: 45, A: 28, S: 10, E: 30, C: 25 }`
- `valuesPriority` (JSONB): Ordered list of core values.
- `interests` (TEXT[]): List of topics/interests.
- `createdAt` (TIMESTAMP, default: now)

### 2.7. `CareerMatch`
Stores individual career suggestions generated by the analysis algorithm for a user, as well as job match metrics.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`)
- `careerName` (VARCHAR)
- `score` (DECIMAL, scale: 2) - Overall compatibility percentage.
- `matchReason` (TEXT) - Explaining why the user's DNA fits this career.
- `requiredSkills` (JSONB) - List of skills needed.
- `skillGaps` (JSONB) - List of skills user does not currently possess.
- `strengths` (TEXT[]) - Core profile assets for this role.
- `weaknesses` (TEXT[]) - Key gaps/liabilities for this role.
- `createdAt` (TIMESTAMP, default: now)

### 2.8. `Roadmap`
The active learning track a user initiates for a specific target career.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`)
- `careerMatchId` (UUID, Foreign Key referencing `CareerMatch.id`, Nullable)
- `title` (VARCHAR)
- `description` (TEXT, Nullable)
- `status` (ENUM: 'ACTIVE', 'COMPLETED', 'ARCHIVED', default: 'ACTIVE')
- `createdAt` (TIMESTAMP, default: now)

### 2.9. `RoadmapMilestone`
Action items within a specific roadmap.
- `id` (UUID, Primary Key)
- `roadmapId` (UUID, Foreign Key referencing `Roadmap.id`)
- `title` (VARCHAR)
- `description` (TEXT)
- `orderIndex` (INTEGER)
- `status` (ENUM: 'NOT_STARTED', 'IN_PROGRESS', 'COMPLETED', default: 'NOT_STARTED')
- `resources` (JSONB) - Curated link cards: title, url, type (e.g., 'course', 'article', 'book')
- `targetDate` (TIMESTAMP, Nullable)

### 2.10. `Resume`
Details of user resumes uploaded to the platform, along with ATS feedback.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`)
- `fileUrl` (VARCHAR) - Link to storage bucket resource.
- `parsedText` (TEXT) - Raw text content extracted.
- `atsScore` (INTEGER, default: 0) - ATS formatting, keywords, structure score out of 100.
- `formattingAnalysis` (JSONB) - Checks for tables, column layout flags, font issues.
- `keywordAnalysis` (JSONB) - Presence/frequency metrics of key keywords.
- `missingSkills` (TEXT[]) - Skills detected as missing from resume compared to target career.
- `improvementSuggestions` (JSONB) - Bulleted improvement checklist.
- `createdAt` (TIMESTAMP, default: now)

### 2.11. `InterviewSession`
Represents an AI mock interview session.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`)
- `careerMatchId` (UUID, Foreign Key referencing `CareerMatch.id`)
- `averageScore` (INTEGER, default: 0) - Average score of answers.
- `feedbackReport` (TEXT) - General report summary.
- `createdAt` (TIMESTAMP, default: now)

### 2.12. `InterviewQuestion`
Individual questions and answers inside an InterviewSession.
- `id` (UUID, Primary Key)
- `sessionId` (UUID, Foreign Key referencing `InterviewSession.id`)
- `type` (ENUM: 'HR', 'TECHNICAL', 'BEHAVIORAL')
- `questionText` (TEXT)
- `userAnswer` (TEXT, Nullable)
- `feedback` (TEXT, Nullable)
- `score` (INTEGER, Nullable) - Question-specific score.
- `orderIndex` (INTEGER)

### 2.13. `CareerDnaScore`
The consolidated score repository calculating values out of 100.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`, Unique)
- `resumeScore` (INTEGER) - Calculated from ATS score.
- `skillsScore` (INTEGER) - Target career skills coverage.
- `projectsScore` (INTEGER) - Combined score of registered projects.
- `certificationsScore` (INTEGER) - Combined score of completed certifications.
- `interviewScore` (INTEGER) - Average score from mock interviews.
- `finalScore` (INTEGER) - Weighted composite score out of 100.
- `updatedAt` (TIMESTAMP, default: now)

### 2.14. `ChatMessage`
Maintains conversational log history for the AI Advisor session.
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key referencing `User.id`)
- `sessionId` (VARCHAR) - Groups messages in a chat thread.
- `sender` (ENUM: 'USER', 'ASSISTANT')
- `content` (TEXT)
- `createdAt` (TIMESTAMP, default: now)

## 3. Prisma Schema Blueprint

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

enum SkillLevel {
  BEGINNER
  INTERMEDIATE
  ADVANCED
}

enum SkillSource {
  MANUAL
  RESUME_PARSED
}

enum RoadmapStatus {
  ACTIVE
  COMPLETED
  ARCHIVED
}

enum MilestoneStatus {
  NOT_STARTED
  IN_PROGRESS
  COMPLETED
}

enum SenderType {
  USER
  ASSISTANT
}

enum CertificationStatus {
  NOT_STARTED
  IN_PROGRESS
  COMPLETED
}

enum InterviewQuestionType {
  HR
  TECHNICAL
  BEHAVIORAL
}

model User {
  id                String             @id @default(uuid()) @db.Uuid
  email             String             @unique
  passwordHash      String?
  createdAt         DateTime           @default(now())
  updatedAt         DateTime           @updatedAt
  profile           Profile?
  assessmentResults AssessmentResult[]
  careerMatches     CareerMatch[]
  roadmaps          Roadmap[]
  resumes           Resume[]
  interviewSessions InterviewSession[]
  careerDnaScore    CareerDnaScore?
  chatMessages      ChatMessage[]
}

model Profile {
  id              String          @id @default(uuid()) @db.Uuid
  userId          String          @unique @db.Uuid
  user            User            @relation(fields: [userId], references: [id], onDelete: Cascade)
  firstName       String
  lastName        String
  avatarUrl       String?
  currentJob      String?
  educationLevel  String?
  experienceYears Int             @default(0)
  updatedAt       DateTime        @updatedAt
  skills          UserSkill[]
  projects        Project[]
  certifications  Certification[]
}

model UserSkill {
  id        String      @id @default(uuid()) @db.Uuid
  profileId String      @db.Uuid
  profile   Profile     @relation(fields: [profileId], references: [id], onDelete: Cascade)
  name      String
  level     SkillLevel  @default(BEGINNER)
  source    SkillSource @default(MANUAL)

  @@index([name])
}

model Project {
  id            String   @id @default(uuid()) @db.Uuid
  profileId     String   @db.Uuid
  profile       Profile  @relation(fields: [profileId], references: [id], onDelete: Cascade)
  title         String
  description   String
  githubUrl     String?
  demoUrl       String?
  techStack     String[]
  strengthScore Int      @default(0)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}

model Certification {
  id             String              @id @default(uuid()) @db.Uuid
  profileId      String              @db.Uuid
  profile        Profile             @relation(fields: [profileId], references: [id], onDelete: Cascade)
  title          String
  issuer         String
  issueDate      DateTime?
  credentialId   String?
  credentialUrl  String?
  relevanceScore Int                 @default(0)
  status         CertificationStatus @default(COMPLETED)
  createdAt      DateTime            @default(now())
  updatedAt      DateTime            @updatedAt
}

model AssessmentResult {
  id             String   @id @default(uuid()) @db.Uuid
  userId         String   @db.Uuid
  user           User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  riasecScores   Json     // e.g. { R: 12, I: 45, A: 28, S: 10, E: 30, C: 25 }
  valuesPriority Json     // Ordered array of values
  interests      String[]
  createdAt      DateTime @default(now())
}

model CareerMatch {
  id                String             @id @default(uuid()) @db.Uuid
  userId            String             @db.Uuid
  user              User               @relation(fields: [userId], references: [id], onDelete: Cascade)
  careerName        String
  score             Decimal            @db.Decimal(5, 2)
  matchReason       String
  requiredSkills    Json               // Array of skill strings
  skillGaps         Json               // Array of missing skill strings
  strengths         String[]
  weaknesses        String[]
  createdAt         DateTime           @default(now())
  roadmaps          Roadmap[]
  interviewSessions InterviewSession[]
}

model Roadmap {
  id            String             @id @default(uuid()) @db.Uuid
  userId        String             @db.Uuid
  user          User               @relation(fields: [userId], references: [id], onDelete: Cascade)
  careerMatchId String?            @db.Uuid
  careerMatch   CareerMatch?       @relation(fields: [careerMatchId], references: [id], onDelete: SetNull)
  title         String
  description   String?
  status        RoadmapStatus      @default(ACTIVE)
  createdAt     DateTime           @default(now())
  milestones    RoadmapMilestone[]
}

model RoadmapMilestone {
  id          String          @id @default(uuid()) @db.Uuid
  roadmapId   String          @db.Uuid
  roadmap     Roadmap         @relation(fields: [roadmapId], references: [id], onDelete: Cascade)
  title       String
  description String
  orderIndex  Int
  status      MilestoneStatus @default(NOT_STARTED)
  resources   Json            // Array of resource objects {title, url, type}
  targetDate  DateTime?
}

model Resume {
  id                     String   @id @default(uuid()) @db.Uuid
  userId                 String   @db.Uuid
  user                   User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  fileUrl                String
  parsedText             String
  atsScore               Int      @default(0)
  formattingAnalysis     Json     // Column, structure flags
  keywordAnalysis        Json     // Found, target keyword checks
  missingSkills          String[]
  improvementSuggestions Json     // Suggested steps
  createdAt              DateTime @default(now())
}

model InterviewSession {
  id            String              @id @default(uuid()) @db.Uuid
  userId        String              @db.Uuid
  user          User                @relation(fields: [userId], references: [id], onDelete: Cascade)
  careerMatchId String              @db.Uuid
  careerMatch   CareerMatch         @relation(fields: [careerMatchId], references: [id], onDelete: Cascade)
  averageScore  Int                 @default(0)
  feedbackReport String
  createdAt     DateTime            @default(now())
  questions     InterviewQuestion[]
}

model InterviewQuestion {
  id           String                @id @default(uuid()) @db.Uuid
  sessionId    String                @db.Uuid
  session      InterviewSession      @relation(fields: [sessionId], references: [id], onDelete: Cascade)
  type         InterviewQuestionType
  questionText String
  userAnswer   String?
  feedback     String?
  score        Int?
  orderIndex   Int
}

model CareerDnaScore {
  id                 String   @id @default(uuid()) @db.Uuid
  userId             String   @unique @db.Uuid
  user               User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  resumeScore        Int      @default(0)
  skillsScore        Int      @default(0)
  projectsScore      Int      @default(0)
  certificationsScore Int      @default(0)
  interviewScore     Int      @default(0)
  finalScore         Int      @default(0)
  updatedAt          DateTime @updatedAt
}

model ChatMessage {
  id        String     @id @default(uuid()) @db.Uuid
  userId    String     @db.Uuid
  user      User       @relation(fields: [userId], references: [id], onDelete: Cascade)
  sessionId String
  sender    SenderType
  content   String
  createdAt DateTime   @default(now())

  @@index([sessionId])
}
```
